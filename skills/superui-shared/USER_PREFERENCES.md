# SuperUI User Preferences

This file is the local fallback for long-lived SuperUI preferences.

Preferred system-level location:

```text
~/.codex/superui/USER_PREFERENCES.md
```

Use this file only when the system-level location is unavailable or not writable.

## Recording Rules

- Record only stable preferences the user states explicitly.
- Do not record one-off project requirements.
- Current user instructions override stored preferences.
- If a preference conflicts with source evidence, DESIGN.md, accessibility, or responsive rules, follow the stronger constraint and record the conflict in the task artifact.

## Entry Template

```markdown
### YYYY-MM-DD - Short preference title
- Source: "exact user wording"
- Scope: visual | layout | component | technology | workflow | forbidden-pattern
- Preference: concise normalized rule
- Confidence: low | medium | high
- Notes: optional evidence or known exceptions
```
